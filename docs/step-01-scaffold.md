# Step 01 Scaffold

## 这一阶段做什么

- 建立项目目录
- 建立核心模型和接口
- 建立 API 最小服务
- 建立前端最小空壳

## 这一阶段不做什么

- 不接数据库
- 不接模型服务
- 不写 LangGraph
- 不做真实检索
- 不做真实桌面能力

## 为什么先做骨架

如果一开始就把数据库、模型、Agent 流程混在一起写，学习成本和返工成本都会很高。  
先把目录、模型、接口和应用边界固定下来，后续每增加一个能力都只是在既有结构上补实现，而不是反复重构。

## 你在这一阶段应该关注什么

- 这些概念为什么属于 domain
- 哪些代码属于 app 壳层
- 为什么接口先于实现
- 为什么 Step 1 故意不引入外部复杂依赖

## 下一步

Step 2 建议从“文档扫描与项目元数据”开始。  
那一阶段会第一次引入真正的业务行为，但仍然不会接向量数据库和 LLM。
